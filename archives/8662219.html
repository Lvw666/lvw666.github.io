JSON 是什么

JSON 是 Ajax 发送和接收数据的一种格式

JSON 全称是 JavaScript Object Notation JavaScript

前后端通信需要用到 JSON, 因为前后端语言不同, 需要有一个统一的数据格式

JSON 有 3 种形式，每种形式的写法都和 JS 中的数据类型很像，可以很轻松的和 JS 中的数据类型互相转换

JSON 的 3 种形式

  1. 简单值形式
  2. 对象形式
  3. 数组形式

JSON 简单值形式

JSON 文件以 .json 结尾

JSON 的简单值形式就对应着 JS 中的基础数据类型

数字、字符串、布尔值、null

JSON 简单值形式 注意事项:

  1. JSON 中没有 undefined 值
  2. JSON 中的字符串必须使用双引号
  3. JSON 中不能注释
  4. JSON 中一次只能写一个值

JSON 对象形式

JSON 的对象形式就对应着 JS 中的对象

JSON 对象形式 注意事项:

  1. JSON 中对象的属性名必须用双引号，属性值如果是字符串也必须用双引号
  2. JSON 中只要涉及到字符串，就必须使用双引号
  3. 不支持 undefined
  4. 一次只能写一个对象
  5. JSON 中不能注释

JSON 数组形式

JSON 的数组形式就对应着 JS 中的数组

注意事项:

  1. 数组中的字符串必须用双引号
  2. JSON 中只要涉及到字符串，就必须使用双引号
  3. 不支持 undefined
  4. JSON 中不能注释

JSON 的常用方法

  1. JSON.stringify()
  2. JSON.parse()

JSON.stringify()

JSON.stringify() 可以将 JS 的基本数据类型、对象或者数组转换成 JSON 格式的字符串

JSON.parse()

JSON.parse() 可以将 JSON 格式的字符串解析成 JS 中的对应值

一定要是合法的 JSON 字符串，否则会报错

xhr 对象属性

responseType 和 response 属性

responseText, 文本形式的响应内容

responseType 处理响应的类型, 默认 text, 此时 response 和 responseText 效果一样

可以设置为 json, 会自动解析成对象, 需要使用 response 查看结果

response 支持的类型更多, 可以替换到 responseText

兼容性: IE6~9 不支持，IE10 开始支持

timeout 属性

设置请求的超时时间（单位 ms）,通常会配合 timeout 事件

兼容性: IE6~7 不支持，IE8 开始支持

withCredentials 属性

指定使用 Ajax 发送请求时是否携带 Cookie

使用 Ajax 发送请求，默认情况下，同域时，会携带 Cookie

跨域时，不会携带 cookie

跨域时，不携带 cookie, 如果需要, 设置 xhr.withCredentials = true;

最终能否成功跨域携带 Cookie，还要看服务器同不同意

兼容性: IE6~9 不支持，IE10 开始支持

xhr 对象方法

abort() 方法

终止当前请求,一般配合 abort 事件一起使用

setRequestHeader() 方法

可以设置请求头信息

只能设置浏览器认为安全的头信息, 我们一般用来设置 Content-type

请求头中的 Content-Type 字段用来告诉服务器，浏览器发送的数据是什么格式的

发送 post 请求时, 需要设置这个头信息

form 表单提交时, 也有对应的属性, enctype

application/x-www-form-urlencoded 是 enctype 属性的默认值

在 ajax 中使用, 传输键=值格式

在 Ajax 中使用, 传输 json

xhr 对象事件

load 事件

响应数据可用时触发, 可以替换 readystatechange, 推荐替换

兼容性: IE6~8 不支持 load 事件

也可以使用 addEventListener

error 事件

请求发生错误时触发

兼容性: IE10 开始支持

abort 事件

调用 abort() 终止请求时触发

兼容性: IE10 开始支持

timeout 事件

请求超时后触发

兼容性: IE8 开始支持
