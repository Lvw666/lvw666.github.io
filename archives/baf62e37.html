组件间传值

父组件传值类似于 html 标签的属性和属性值

子组件接收值, 需要使用 props 属性

静态传值和动态传值

通过 属性="属性值" 这种方式来传值, 数据类型是 string, 属于静态传值

可以通过绑定属性的方式进行传参, 这种形式属于动态传值

传值校验

父组件传值, 子组件接收值, 那么子组件在接收值的时候, 可以对值进行一些校验

比如可以校验数据类型, 支持的数据类型有 String, Boolean, Array, Object, Number, Function ...

子组件需要父组件传值, 但是父组件没有传值, 默认是 undefined

可以使用 required, 校验必传

如果没有传值, 可以设置默认值

默认值也可以是一个函数

也可以使用 validator, 自定义校验规则, 比如必须小于 1000

传值时的命名方式

传值时, 写在 html 代码中的命名应该使用中划线的方式, 比如content-data

与之对应的, 在 js 代码中的命名应该使用小驼峰, 比如contentData, 这样才能成功接收到值

单向数据流

从父组件向子组件传值时, 子组件可以修改父组件的传过来的值吗?

点击会报警告

提示不能修改 count, 因为这是父组件传过来的数据, 是只读的 readonly

这就是单向数据流, 父组件传递过来的数据, 子组件只能使用, 不能修改

为什么会有单向数据流的规定呢?

因为父组件不止向一个子组件传值, 如果其中某个子组件修改了这个值, 那么其他子组件再使用这个值的时候, 就会受影响, 就没有办法实现组件数据的相互独立

如果想修改怎么办呢? 可以先用一个变量把这个值存起来, 然后再修改
