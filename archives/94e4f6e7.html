什么是 class

class ==> 类

人类：类

具体的人：实例、对象

类可以看做是对象的模板，用一个类可以创建出许多不同的对象

定义 class

类名一般首字母大写

添加构造方法

实例化对象时, 自动执行构造方法，所以必须有构造方法，但可以不写出来

class 实例化对象

使用 new 关键字

实例化时传值

定义方法

类属性/方法

属性 = 属性值

前面不加 var const let

方法(){}

类的静态属性/方法

不需要实例化, 就可以使用类直接调用的方法/属性

使用关键字 static(静态)

静态方法/属性只能类来调用, 对象不能调用

静态属性/方法和实例化的属性/方法, 不冲突

静态属性和静态方法的添加另一种添加方式

类.静态属性 = 属性值

类.静态方法 = function(){}

类的静态方法, this 指向类

实例的方法, this 指向实例

类的私有属性/方法

一些属性和方法, 只能在类的内部调用, 不能在类的的外部调用

一般情况下，类的属性和方法都是公开的

公有的属性和方法可以被外界修改，造成意想不到的错误

模拟私有属性和方法如下：

  • 使用_开头, 表示私有(约束性不强)

将私有属性和方法移出类

extends

使用 extends 子类可以继承父类的属性或方法

子类改写继承的属性或方法

同名覆盖

子类新增自己的属性或者方法

this 必须在 super 下面

super

作为函数使用, 代表父类的构造方法, 只能用在子类的构造方法中, 用在其他地方会报错

super 虽然代表了父类的构造方法, 但是内部的 this, 指向子类的实例

作为对象使用

如果在构造方法中使用, 代表父类的原型对象 Person.prototype

通过 super 调用父类的方法时, 方法内部的 this, 指向当前类的子类的实例

借助 super, 可以在父类的方法上做修改, 而不是覆盖重写

如果 super 用在静态方法中, 代表父类, 而不是父类的原型

如果 super 用在静态方法中, 里面的 this 指向子类

super 的注意事项
