用 new 操作符来调用函数

新的函数调用方式

JS 规定，使用 new 操作符调用函数会进行“四步走”：

  1. 函数体内会自动创建出一个空白对象
  2. 函数的上下文（this）会指向这个对象
  3. 函数体内的语句会执行
  4. 函数会自动返回上下文对象，即使函数没有 return 语句

四步走详解

第 1 步：函数体内会自动创建出一个空白对象

第 2 步：函数的上下文（this）会指向这个对象

第 3 步：执行函数体中的语句

第 4 步：函数会自动返回上下文对象，即使函数没有 return 语句

构造函数

什么是构造函数

用 new 调用一个函数，这个函数就被称为构造函数，任何函数都可以是构造函数，只需要用 new 调用它

顾名思义，构造函数用来构造新对象，它内部的语句将为新对象添加若干属性和方法，完成对象的初始化

构造函数必须用 new 关键字调用，否则不能正常工作，正因如此，开发者约定构造函数命名时首字母要大写, 可以和普通函数(小写字母开头)区分开

构造函数命名规范

构造函数名称首字母约定大写

一定要记住：

一个函数是不是构造函数，要看它是否用 new 调用，而至于名称首字母大写， 完全是开发者的习惯约定

构造函数的调用方式

使用 new 调用构造函数

如果不使用 new, 就会变成普通函数, 返回 undefined

构造函数中的 this

构造函数中的 this 不是函数本身, 而是函数体内会自动创建出一个空白对象

原型和原型链

什么是 prototype

面试常考, 出现率 90%

任何函数都有 prototype 属性，prototype 是英语原型的意思

prototype 属性值是个对象，它默认拥有 constructor 属性指回函数

普通函数来说的 prototype 属性没有任何用处，而构造函数的 prototype 属性非常有用

构造函数 的 prototype 属性 是 构造函数的实例 的 原型

翻译一下: People.prototype 是 xiaoming 的 原型

原型链查找

JavaScript 规定：实例可以打点访问它的原型的属性和方法

有些属性和方法, 实例没有, 但是实例的原型有, 实例就可以通过打点调用的方式, 使用这些属性和方法

实例没有, 实例的原型有, 一样能用

如果实例的原型也没有呢? 实例的原型也有原型, 可以去找实例的原型的原型

这被称为原型链查找

如果实例本身就有相关的属性或者方法, 就不会进行原型链查找

在原型上添加方法或属性

在之前的课程中，我们把方法都是直接添加到实例身上

这三个sayHello(), 是同一个函数吗?

把方法直接添加到实例身上的缺点：

每个实例和每个实例的方法函数都是内存中不同的函数，造成了内存的浪费

解决办法：将方法写到prototype上

实际上, 也可以用这种方式, 添加每个对象都有的公有属性

hasOwnProperty()方法

hasOwnProperty 方法可以检查对象是否真正自己拥有某属性或者方法

in

in 运算符只能检查某个属性或方法是否可以被对象访问，不能检查是否是自己的属性或方法
