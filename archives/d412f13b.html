模块和模块化

模块, 经过封装的 js 文件, 里面的功能可以拿来即用

python 中也有模块的概念

js 模块化在项目中的应用

我们通常会把一个复杂的功能拆分成一个一个的小块, 最后组装起来, 形成完整的功能

为什么这么做呢?

因为所有的 JS 代码都可以写到一个文件里, 但是这样代码量很大, 不易于维护, 所以需要按功能, 按逻辑, 进行拆分, 提升开发效率, 降低维护成本

当然, 依然会存在一些问题

  1. 全局变量污染
  2. 模块加载顺序问题

所以我们需要模块化规范来解决, 让我们更方便的来使用模块化进行编程

在不同的位置, 遵守的规范不同, 也就是说, 我们现在浏览器端和服务端有两种规范

温馨提醒: 模块化需要有服务器的支持, 我们需要安装 live server 插件, 来创建一个小型的服务器

导入模块

一个模块即使没有数据导出, 也可以导入

导入模块的效果, 就是把模块中的代码执行一遍, 仅执行一遍

引入 js 文件时的注意事项:

如果 js 文件作为模块导入, 需要加上type="module"

import 和 import()

import, 会提升到代码的第一行执行

import 和 export 命令只能在模块的顶层(最外层), 不能在代码块中执行

如果需要按需导入, 使用 import()
