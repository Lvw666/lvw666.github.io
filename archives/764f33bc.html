Webpack 是什么?

Webpack 是现代 JavaScript 应用程序的一个静态模块打包工具(module bundler)

什么是静态资源?

开发过程中存在于本地的 js/css/图片/图标字体等文件，就是静态的

动态的内容，Webpack 没办法处理，只能处理静态的

Webpack 的作用

  1. 将多个不同类型文件打包
  2. 编译代码，确保浏览器能解析
  3. 对源代码进行格式化校验
  4. 对代码进行压缩，减少文件体积，提高加载速度
  5. 把多个文件合并成一个文件，减少 http 请求的次数，提高生产环境的运行效率
  6. 有热更新功能，提高开发效率
  7. 不同环境，提供不同的打包策略

Webpack 的核心配置

入口（Entry）

  • 告诉 webpack 应该使用哪个模块，来作为构建整个项目的开始
  • 默认src/index.js

出口（Output）

  • 告诉 webpack 在哪里输出它所创建的 bundle，以及如何命名这些文件
  • 默认dist/main.js

加载器（Loader）

  • webpack 只能理解 JS 和 JSON 文件，loader 让 webpack 能够去处理其他类型的文件，并将它们转换为有效模块，以供应用程序使用, 充当一个翻译的角色
  • 命名方式, xxx-loader, 比如css-loader | html-loader , 翻译 css 和 html, 让 webpack 可以识别 css 和 html 文件
  • 专门处理一类的文件

插件（Plugins）

  • 插件是 webpack 的支柱功能，能够执行范围更广的任务，包括：打包优化、压缩、重新定义环境中的变量等
  • 实现 loader 之外的其他功能
  • 命名方式xxx-webpack-plugin, 比如 html-webpack-plugin

模式（Mode）

  • 区分环境，不同的环境打包逻辑不同
  • development (优化打包速度，提供调试的辅助功能)
  • production (优化打包结果，打包之后代码的运行效率和性能优化)
  • none (运行最原始的打包, 不做任何额外处理)

编译 JS 代码 - 使用 Babel

Babel 是什么?

Babel 是 JavaScript 的编译器，用来将 ES6 的代码，转换成 ES6 之前的代码

中文网站: Babel中文文档

英文网站: Babel英文文档

Babel 本身可以编译 ES6 的大部分语法，比如 let、const、箭头函数、类

但是对于 ES6 新增的 API，比如 Set、Map、Promise 等全局对象，以及一些定义在全局对象上的方法（比如 Object.assign/Array.from）都不能直接编译，需要借助其它的模块

Babel 一般需要配合 Webpack 来编译模块语法

Babel 配置文件由很多种写法：

  • babel.config.js
  • babel.config.json
  • .babelrc
  • .babelrc.js
  • .babelrc.json
  • package.json 中 babel：不需要创建文件，在原有文件基础上写

具体的配置规则, 以 babel.config.js 配置文件为例：

  • @babel/preset-react：一个用来编译 React jsx 语法的预设
