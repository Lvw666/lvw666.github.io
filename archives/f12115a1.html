CSS 盒模型

发表于2022-04-22|更新于2022-04-22|Web 前端☛ CSS
|字数总计:4.5k|阅读时长:21分钟|阅读量:

所有 HTML 标签都可以看成矩形盒子，由width、height、padding、border,margin构成，称为盒模型

width、height属性

width 属性表示盒子内容的宽度

width 属性的单位通常是 px，移动端开发也会涉及百分数、rem 等单位

数字一般就是 20px, 30px…

百分比就是 50%, 60%, 相对于父元素

auto 表示自动, 就是默认宽度

当块级元素（div、h 系列、li 等）没有设置 width 属性时，它将自动撑满(父元素的 100%)

width 属性不可继承

height 属性表示盒子内容的高度

height 属性的单位通常是 px，移动端开发也会涉及百分数、rem 等单位

盒子的 height 属性如果不设置，它将自动被其内容撑开，如果没有内容，则 height 默认是 0

内边距 padding

padding 属性详解

padding 是盒子的内边距，即盒子边框内壁到文字的距离

padding ==> 内边距

margin ==> 外边距

因为内心脆弱, 所以经常会怕(pa)

外表彪悍的人, 喜欢骂(ma)人

padding 是四个方向的

四个方向的 padding，可以分别用小属性进行设置

小属性 含义
padding-top 上 padding
padding-right 右 padding
padding-bottom 下 padding
padding-left 左 padding

padding 的四数值写法

padding 属性如果用四个数值以空格隔开进行设置，分别表示上、右、下、左的 padding

padding 的三数值写法

padding 属性如果用三个数值以空格隔开进行设置，分别表示上、左右、下的 padding

padding 的二数值写法

padding 属性如果用二个数值以空格隔开进行设置，分别表示上下、左右的 padding

外边距 margin

margin 属性详解

margin 是盒子的外边距，即盒子和其他盒子之间的距离

margin 也有四个方向

小属性 含义
margin-top 上 margin，”向上踹”
margin-right 右 margin，”向右踹”
margin-bottom 下 margin，”向下踹”
margin-left 左 margin，”向左踹”

margin 的塌陷

如果有两个元素, margin:20px, 那这两个元素之间的距离应该是多少呢?

本能的, 我们觉得应该是第一个元素的margin-bottom加上第二个元素的margin-top

竖直方向的 margin 有塌陷现象

小的 margin 会塌陷到大的 margin 中, 从而 margin 不相加，只以大值为准

一些元素有默认的margin

一些元素（比如 body、ul、p 等）都有默认的 margin，在开始制作网页的时候，要将他们清除

清除默认样式

盒子的水平居中

将盒子左右两边的 margin 都设置为 auto，盒子将水平居中

文本居中是 text-align: center; 和盒子水平居中 margin: 0 auto;是两个概念

盒子的垂直居中，需要使用绝对定位技术实现

div 和 span

div 标签和 span 标签都可以理解成是空盒子, 可以放其他元素或者标签进去

区别在于 div 独占一行(块级元素), span 一行可以有很多个(内联元素)

清除div中图片间的缝隙

方法如下:

  1. 定义font-size="0"
  2. 将图片都定义为往同一个方向浮动 float: left; 或 float: right;

可以同时设置, 边框粗细, 边框样式, 颜色

边框的单独设置

只给盒子的某个方向单独设置边框

属性值支持连写

border-radius 圆角

radius 半径

如果 border-radius, 超过宽度的一半, 正方形, 会变成正圆

如果想把正方形变成圆, 只需要设置 border-radius 的值等于宽度的一半即可

盒子总宽高问题

边框会撑大盒子

如果设置了宽高, 又设置了边框, 你会发现, 盒子变大了…

我们可以得到如下结论:

盒子宽度 = 左边框+内容宽度+右边框

盒子高度 = 上边框+内容高度+下边框

盒子的总宽度一般都是计算好的, 如果因为边框导致盒子变大, 会影响整体布局, 现阶段的解决方案, 重新计算, 把边框算进去, 缩小宽高的数值

内边距把盒子撑大

可以通过浏览器很方便的调试

双击定位数字, 上箭头往上加, 下箭头往下减

内边距会把边框撑大

盒子实际大小终极计算公式

盒子宽度 = 左边框 + 左 padding + 内容宽度 + 右 padding + 右边框

盒子高度 = 上边框 + 上 padding + 内容宽度 + 下 padding + 下边框

现阶段的解决方案:

自己计算多余大小，手动在内容中减去（手动内减）

width、height非总宽高

盒子的总宽度 = width + 左右padding + 左右border

盒子的总高度 = height + 上下padding + 上下border

解决方案: box-sizing

之前为了防止盒子被撑大, 需要人工计算内容区的宽高。

如果总宽度确定, 边框和 padding 越大, 内容区自动缩小来适应不可以吗?(往里面挤)

这时候就可以用box-sizing来解决。

盒子的类型

content-box

标准盒模型: content-box

盒子的默认类型，设定的是内容区的宽高，不是盒子的总宽高。

border-box

怪异盒模型: border-box

盒子的另一种类型，设定的是盒子的总宽高，会自动根据所设定的总宽高来调整内容区的大小。

overflow 溢出

内容超出盒子范围怎么办?

visible

溢出不做处理

hidden

超出部分隐藏, 没有滚动条

scroll

超出部分隐藏, 有滚动条

auto

按需出现滚动条

box-shadow 阴影

box 盒子 shadow 阴影

支持外阴影, 内阴影, 多阴影(逗号隔开)

阴影不占用空间

测试地址: https://www.w3school.com.cn/tiy/c.asp?f=css_box-shadow
