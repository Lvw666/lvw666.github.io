节点关系简介

node 节点

child 儿子

parent 父亲

first 第一个

last 最后一个

previous 上一个

next 下一个

sibling 兄弟

关系 考虑所有节点
子节点 childNodes
父节点 parentNode
第一个子节点 firstChild
最后一个子节点 lastChild
前一个兄弟节点(找哥哥) previousSibling
后一个兄弟节点(找弟弟) nextSibling

找节点的方法

DOM 中，文本节点也属于节点，在使用节点的关系时一定要注意

在标准的 W3C 规范中，空白文本节点也应该算作节点(空格也算一个节点)

排除文本节点的干扰: 只考虑元素节点的属性

后一个兄弟节点 nextSibling nextElementSibling

改变元素节点中的内容

改变元素节点中的内容可以使用两个相关属性：① innerHTML ② innerText

innerHTML 属性能以 HTML 语法获取/设置节点中的内容

innerText 属性只能以纯文本的形式获取/设置节点中的内容

纯文本, 没有区别

如果有标签, 就会有区别

有标签, innerHTML 会解析, innerText 会原样输出

删除节点

removeChild() 方法从 DOM 中删除一个子节点

节点不能主动删除自己，必须由父节点删除它

克隆节点

cloneNode() 方法可以克隆节点，克隆出的节点是“孤儿节点”

参数是一个布尔值，表示是否采用深度克隆

如果为 true，则该节点的所有后代节点也都会被克隆，如果为 false(默认)，则只克隆该节点本身
