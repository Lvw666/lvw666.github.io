媒体查询

响应式布局

2010 年 5 月由 EthanMarcotte 提出的一个概念

一个网站兼容多种终端

对不同尺寸的屏幕做出响应（媒体查询），并进行相应布局

示例：https://www.samsung.com/cn/

媒体查询

(Media querys), 简称媒询

一套样式很难适应各种大小的屏幕

媒体查询针对各种大小的屏幕写样式，让我们的页面在不同大小的屏幕上都能正常显示

媒体类型

  1. all（所有设备 默认值）

  2. screen（屏幕设备）

  3. print（打印设备）

  4. speech（屏幕阅读器，一般供残障人士使用）

all 和 screen 比较常用

媒体查询中的逻辑

与( and ) 或( , ) 非( not )

与( and ), 查询条件全部为真时生效

或( , ) 查询条件中的任意一个为真时生效

或( , ) 可以把查询条件分开

非( not ) 对当前查询条件取反

当 not 与 and 同时出现，not 对整个媒体查询生效

not 与逗号分隔的多个媒体查询同时存在时，not 只对它所在的那个查询生效

媒体特性

宽度：width / max-width / min-width

dpr：-webkit-device-pixel-ratio / -webkit-max-device-pixel-ratio / -webkit-min-device-pixel-ratio

屏幕方向 orientation：landscape / portrait

width 代表只有完全等于这个宽度才生效

dpr 屏幕像素比, 加上 -webkit-, 是因为只有支持 webkit 内核的浏览器才生效

屏幕方向 orientation：landscape(横屏) / portrait(竖屏)

使用 js 获取 dpr 的方法

断点的设置

断点设置多少合适呢?

  1. 业界主流的做法

  1. 改变屏幕大小，当页面显示不正常（或不符合要求）的时候，就需要设置断点了

媒体查询的书写位置

1.样式表（style 标签或单独的 CSS 文件）中（推荐）

2.样式外链 link 中（不推荐）

注意：论媒体查询的条件是否满足，都会下载样式文件

媒体查询的策略

  1. 无策略
  2. PC 端优先
  3. 移动端优先

参考 bootstrap 的断点

无策略

PC 端优先, 从大到小写

移动端优先, 从小到大写
