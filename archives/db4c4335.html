Vuex

专门在 Vue 中实现集中式状态（数据）管理的一个 Vue 插件，对 vue 应用中多个组件的共享状态进行集中式的管理（读/写），也是一种组件间通信的方式，且适用于任意组件间通信。

什么时候使用 Vuex

  1. 多个组件依赖于同一状态
  2. 来自不同组件的行为需要变更同一状态
  3. 说的通俗点, 就是共享

Vuex 原理

下面这张, 是官方文档的 Vuex 原理图

从图上可以看出 Vuex 有三个重要组成部分

state 里面存数据, 类型是 object

组件中如果需要修改数据, 需要先触发派发逻辑(dispatch)

往下走会到 actions, actions 的数据类型也是对象

需要在 actions 中自己调用 commit, 会走到 mutations, mutations 也是一个对象

mutations 里面的 function, 可以拿到state和传过来的123

修改 state 的值, 会自动走 mutate, 去修改 state, 同时会把最新的数据, 渲染到页面

可能会有人觉得, actions 多余, 直接调用 mutations 也可以

确实可以, 如果不需要 ajax, 可以直接通过 commit, 来触发 mutations

如果需要 ajax 请求数据, 则需要走 actions

如果牵涉到相关的判断逻辑, 也需要写到 actions 里面

devtools, 是开发者工具, 主要和 mutations 进行对话, 因为 mutations 包含了最主要的逻辑

跟我们去饭店吃饭类似

如果经常吃, 跟后厨比较熟, 菜谱都清楚, 也可以直接联系后厨, 除非菜谱变了, 才需要找服务员

另外还有一个隐藏角色 store, actions, mutations, state, 都要靠 store 来管理

改变全局变量必须遵循一定的规则

  1. 派发(dispatch)一个 action, 告诉 Vuex, 我接下来要做什么事情
  2. store 感知到触发了一个 changeName(action 的名字)的 action, 执行 changeName
  3. 提交一个 commit, 触发一个 mutation(变化)
  4. 对应的 mutation 会执行
  5. 在 mutation 中修改数据

如果牵涉到异步代码, 比如 ajax, 上面的步骤不能省略, 通常 actions 负责异步代码, mutation 负责同步代码
