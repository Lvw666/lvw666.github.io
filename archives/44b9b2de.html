ECMAScript简介

ECMAScript 6 是什么

ECMAScript 是语言的标准, 6 是版本号

ES6 = ECMAScript 这门语言的第 6 代标准

什么是 ECMA

ECMA 欧洲计算机制造商协会, 是一个标准化组织

ECMAScript = 由 ECMA 这个标准化组织制定的一个语言标准

ECMAScript 包括哪些内容

ECMAScript 包括 语法 和 API

ECMAScript 的命名方式

按版本号或者按时间命名

ES6 vs ES2015

ES6->ES7->ES8->… = ES6+

ES2015->ES2016->ES2017->…

ES 与 JavaScript 的关系

JavaScript(浏览器端) = ECMAScript(语法+API) + DOM + BOM

ES6 的兼容性

主流浏览器的最新版本几乎全部支持 ES6

IE 老版本等不支持的浏览器，可以用 Babel 转码

放心大胆地使用 ES6

let 和 const

let 用来声明变量, const 用来声明常量

变量和常量的区别

变量, 值会发生改变的量, 比如年龄

常量, 值不发生改变的量, 比如性别

变量初始化后, 可以重新赋值

常量初始化后, 不可以重新赋值

为什么需要 const

如果需要定义一个变量, 它的值不变, 之前我们除了小心不要重新赋值, 没有更好的办法

const 用于, 一旦初始化, 不希望重新赋值的情况

有了 const, 不需要来回的检查, 如果后面不小心赋值, 浏览器会报错

何时用 const,何时用 let

如果一开始就可以确定是变量, 就用 let

如果一开始不确定, 先用 const, 如果需要重新赋值, 再改回 let

重复声明

所谓重复声明, 就是已经存在的变量或者常量, 再次声明

var 支持, let const 不支持

变量提升

var 会提升变量的声明到当前作用域的顶部, 允许先调用, 后赋值

let const 不存在变量提升

块级作用域

块级作用域是什么

块级作用域是 ES6 中新增的一个作用域

指在大括号{}里面使用 let 或 const 关键字声明变量或常量，就会形成一个块级作用域。

但有两个需要特殊记忆，函数和对象的大括号{}不属于块级作用域。

var 没有块级作用域, 具有块作用域的结构

  1. 块级作用域是指程序中使用花括号 {} 创建的一段代码区域，在这个区域内定义的变量只在该区域内部有效，超出该区域就无法访问。
  2. 常见的块级作用域包括if语句、for循环等。在ES6之前，JavaScript中只有全局作用域和函数作用域，ES6引入了let和const关键字，使得在块级作用域中定义变量成为可能。

模板字符串

模板字符串是什么

普通字符串, 使用单引号''或者双引号""

模板字符串, 使用反引号``

模板字符串语法：${},花括号里填入表达式

  1. 模板字符串是ECMAScript 6（ES6）引入的一种新的字符串类型，使用反引号来定义字符串，可以在其中插入变量、表达式或函数调用等动态的内容，并且支持换行、缩进和多行字符串等特性。
  2. 使用${}可以将任何JavaScript表达式嵌入到模板字符串中，例如${name}可以插入变量name的值，${1+1}可以计算1+1的结果并插入到模板字符串中。模板字符串提供了一种更加灵活、简洁和易于理解的方式来处理动态文本生成的问题。

区别在于混合输出

箭头函数中的 this

this 的指向和函数的定义无关, 和调用有关

规则 上下文
对象.函数() 对象
函数() window
数组[下标]() 数组
IIFE (function(){})() window
定时器, 延时器 window
DOM 事件处理函数 绑定的 DOM 元素
call, apply 自定义
用new调用函数 秘密创建出来的空白对象

箭头函数稍有不同

  • 箭头函数没有自己的 this、arguments，箭头函数里的 this 用的是 parent 作用域的 this。
  • 箭头函数里的 this 是词法作用域，是由代码结构决定的，跟在哪里调用无关，与箭头函数在代码里的位置有关。
  • 确定箭头函数里 this 的指向，可以静态分析代码结构，一直往上找 function，如果有，就是最近的 function 里的 this，如果到最外层都没有，就是 window。
  • 箭头函数的 this，一旦确定后是不可更改的。

记住，箭头函数里的 this 是可以静态分析的，但是函数里的 this 又完全不一样了
