Git 简介

|字数总计:350|阅读时长:1分钟|阅读量:

Git 是什么

分布式, 版本控制, 软件

版本控制

版本: 文件的变化

版本控制: 记录文件变化, 生成版本, 可以查看历史版本, 恢复到指定版本

手动版本控制

手动版本控制(拷贝、改名)

本地版本控制

本地版本控制, 适合单人, 不适合协作, 代表 RCS

集中式版本控制

集中式版本控制, 可以解决协作问题, 但是无法实现本地版本控制, 代表 SVN

分布式版本控制

分布式版本控制, 本地远程都可以做版本控制, 代表 Git

Git 三大区域

Git 监控的是文件的变化

工作区

  • 项目目录，写代码的地方
  • 新增文件、删除文件、修改文件内容

暂存区

  • 暂时存放未提交的内容，打算提交，但还没有提交
  • 只要工作区的文件有变动，就需要将工作区的变动添加到暂存区

Git 库/版本库

  • Git 仓库 = 本地仓库 + 远程仓库
  • 将暂存区内容提交到本地仓库，生成一个版本
  • 如有需要，可将本地仓库中的版本推送至远程仓库

生成版本

工作区和暂存区内容不一致, 可以使用 git add

暂存区和本地仓库内容不一致, 可以使用 git commit
