移动端开发基础

移动Web开发

针对移动端的 Web 开发

Web开发

  1. 使用 HTML, CSS, JavaScript
  2. 代码在浏览器中运行

移动端和PC端

移动端: 手机, 平板

PC 端: 台式机, 笔记本电脑

移动端Web开发和PC端Web开发的关系

  1. 使用的技术都是 HTML, CSS, JavaScript
  2. 移动端 Web 开发, 代码在移动端(手机, 平板)运行
  3. PC 端 Web 开发, 代码在 PC 端(台式机, 笔记本电脑)运行

基本技术体系没有太大的区别, 使用的技术都是 HTML, CSS, JavaScript

不同终端设备具有不同的特性

  1. 屏幕大小不一样, PC 屏幕大, 手机屏幕小
  2. 交互方式不一样, PC 是键盘鼠标, 手机靠手指触摸和滑动
  3. 网络环境和设备性能不同, PC 性能更好, 网速更快, 手机性能差一些, 网速如果不是 wifi 也会是问题
  4. 兼容性不同, PC 端有 IE 浏览器, 手机端没有, 可以用一些更新的技术

App 开发

App 是 application 的缩写

现在实现 App 有三种方式

  1. Native App, 需要安装 App, 程序员需要两个, 安卓苹果, 各一个
  2. Web App ✔️ 使用浏览器打开一个移动端的网站, 看起来和 App 很像
  3. Hybrid App (混合式 App) uniapp, 前端来写, 需要安装 App, 苹果安卓都能跑

分辨率

物理像素

物理像素, 也称设备像素, 就是下图中的一个一个的点

物理像素: physical pixel

设备像素: DP: device pixel

CSS 像素

也叫 逻辑像素（logical pixel）

也叫 设备独立像素（DIP：device independent pixel）

css 像素就是我们实际开发时写的像素, 会自动换算成对应的物理像素

设备像素比

设备像素比（DPR：device pixel ratio）

dpr = (一个方向上)设备像素 / CSS 像素（缩放比是 1 的情况下）

dpr = 2 表示 1 个 CSS 像素用2*2个设备像素来绘制

标清屏和高清屏

高清屏, 一个 css 像素, 需要用更多的物理像素去显示, 所以画面更细腻

缩放

用户通过手指, 放大或者缩小页面上的内容

缩放改变的是 CSS 像素的大小, 物理像素不能改变

缩放分为放大和缩小

放大

缩小

PPI 与 DPI

表示每英寸的物理像素点

PPI: pixels per inch

DPI: dots per inch

viewport

viewport基本概念

使用手机访问 PC 页面, 才有 viewport 的概念, 是专门为移动 web 开发设计的

pc 屏幕大, 手机屏幕小, pc 端的页面, 对于手机来说太大了, 会出现显示不全的情况

viewport 有三个概念, 布局视口, 视觉视口, 理想视口

页面在布局视口中进行布局, 可以超出手机的屏幕范围

页面会缩放到等于手机屏幕的宽度, 保证不出现横向滚动条

布局视口和视觉视口相等, 就是理想视口, 因为页面不会进行缩放

因为一般 PC 端页面都会大于手机屏幕, 所以需要专门针对移动端做一些布局

设置 viewport

我们使用的代码段中, 自带 viewport

viewport 在 meta 标签中设置, 设置内容写在 content 属性中, 格式是名称=值, 多个值用逗号分开

width 设置布局视口的宽度, initial-scale 代表初始缩放值

我们希望布局视口的宽度==视觉视口的宽度,因为每个设备的宽高不一样, 我们不能写死, 所以写成width=device-width, 设备宽度

initial-scale=1.0和width=device-width作用一样, 都是为了不让页面缩放, 都写是为了兼容性

真机调试

要保证手机和电脑在同一网络中

  1. 使用 live server 打开页面

  1. 获取本机 ip, cmd 命令行输入 ipconfig

  1. 替换网址, 替换成http://你的ip地址:5500/viewport.html

  1. 网址转二维码 https://cli.im/

  1. 手机扫二维码, 查看网页

开发中常用单位

px

px, 绝对单位

当需要设置固定宽高时可用，也可用于设置字体大小

%

%, 相对单位

可以用来布局, 可以用来设置宽度，流体布局中会用到

em

em, 相对单位

1em = 自身字体的大小, 一般用来控制行首的缩进，一般不用来布局

有个例外, 设置字体大小时，1em = 父元素字体的大小, 其实这是错觉, 因为字体大小一开始继承的是父级

rem

rem(root element), 相对单位

1rem = 根元素（html）的字体大小, 可以用来布局

vw / vh / vmax / vmin

vw/vh/vmax/vmin, 可以用来布局, 相对单位，视口单位

1vw = 视口宽度的 1%

1vh = 视口高度的 1%

vmin：当前 vw 和 vh 中较小的一个值

vmax：当前 vw 和 vh 中较大的一个值
