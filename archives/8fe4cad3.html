CSS 有哪些基本选择器？它们的权重是如何表示的？

CSS 有以下基本选择器：

  1. 元素选择器：通过 HTML 元素的标签名来选择元素，如 p、div、h1 等。
  2. 类选择器：通过 HTML 元素的 class 属性来选择元素，以 . 开头，如 .main、.nav、.header 等。
  3. ID 选择器：通过 HTML 元素的 id 属性来选择元素，以 # 开头，如 #header、#sidebar、#footer 等。
  4. 属性选择器：通过 HTML 元素的属性来选择元素，格式为 [属性名称*=属性值]，如 [href*=example]。
  5. 伪类选择器：通过 HTML 元素的状态或位置来选择元素，如 :hover、:first-child、:nth-child 等。

选择器的权重表示为一个四位数字，分别对应样式规则中使用的元素选择器、类选择器、ID 选择器和其他选择器的个数。对于每个选择器的权重，可以根据以下规则计算：

  1. 每个元素选择器的权重为 (0, 0, 1)，每个类选择器的权重为 (0, 1, 0)，每个 ID 选择器的权重为 (1, 0, 0)，每个内联样式（style 属性）的权重为 (1, 0, 0, 0), !important 的权重最大。
  2. 当多个选择器叠加时，权重会根据上述规则相加，例如，优先级为 p.class#id 的选择器，其权重为 (1, 1, 1)。
  3. 若多个选择器的权重相同，则后面的规则会覆盖前面的规则。

什么是 BFC？BFC 如何触发？

BFC（块级格式化上下文）是一种 Web 页面布局的 CSS 渲染模式。在 BFC 中，元素按照特定规则布局和渲染，规则包括如下：

  1. BFC 内部的元素垂直排列：块级元素在 BFC 中垂直地放置，一个块级元素顶部和底部都会和其父元素相邻的内容分隔开来。
  2. BFC 内部的元素不会重叠：BFC 内的元素和外界元素不会重叠。
  3. BFC 内部浮动元素也会参与布局：BFC 会包含浮动元素。
  4. BFC 可以防止浮动元素影响到 BFC 外部的元素：BFC 可以创建一个独立的渲染环境，防止浮动元素影响到 BFC 外部的元素。

通常情况下，BFC 可以通过以下方式触发：

  1. 根元素默认为 BFC。
  2. 通过设置元素的 display 属性值为 inline-block、table-cell、flex 等触发 BFC。
  3. 通过设置元素的 float 属性值为 left、right。
  4. 通过设置元素的 position 属性值为 absolute 或 fixed。 触发 BFC 可以带来一些布局和渲染的优势，帮助开发者解决许多常见的页面布局问题。

CSS 的哪些样式可以继承？哪些不可以继承？

在 CSS 中，有些样式可以被子元素继承，而有些则不可以被继承。以下是一些常见的可以继承和不可以继承的 CSS 样式

可以继承的 CSS 样式：

  1. 字体系列：font-family、font-size、font-weight、font-style 等。
  2. 文本系列：text-indent、text-align、text-transform、line-height 等。
  3. 颜色系列：color、background-color、border-color 等。
  4. 列表系列：list-style-type、list-style-position 等。
  5. 其他：cursor、visibility 等。

不可以继承的 CSS 样式：

  1. 盒模型系列：width、height、margin、padding、border 等。
  2. 定位系列：position、top、left、right、bottom 等。
  3. 浮动系列：float、clear。
  4. 生成内容系列：content、quotes。
  5. 其他：display、overflow、z-index 等。

通过了解哪些样式可以继承，哪些不能被继承，我们在页面的样式设计和继承上更加灵活，能够更好地掌握样式继承的规律。

块级元素和行内元素的区别是什么？

块级元素和行内元素是 HTML 中常见的两种元素类型，它们有以下的区别：

  1. 表现形式不同

块级元素默认情况下会占据一行或多行，并且会充满父元素的宽度，而行内元素一般会沿着文本流排列在一行中，并且宽度默认由内容撑开。

  1. 内容模型不同

块级元素可以包含其他块级元素和行内元素，而行内元素只能包含其他行内元素和文本。

  1. 样式表现不同

块级元素可以设置宽度、高度、边框、内边距等样式，并且可以通过 CSS 属性控制块级元素的位置和布局，而行内元素只能设置部分样式，如 color、font-size、font-weight 等。

  1. 默认排列方式不同

块级元素一般从上到下依次排列，而行内元素在文本流中沿着基线向左依次排列。

em 和 rem 的区别是什么？

em 和 rem 都是 CSS 中的长度单位，它们的主要区别在于相对于基准的不同：

  • em：是相对于父元素的 font-size 大小来计算的。例如，如果当前元素的 font-size 为 16px，同时父元素的 font-size 为 20px，那么子元素设置字体大小为 1.5em 时，将会计算为 20px * 1.5 = 30px。
  • rem：是相对于根元素（即 html 元素）的 font-size 大小来计算的。例如，如果根元素的 font-size 设置为 16px，同时一个后代元素设置字体大小为 1.5rem，那么它的实际大小将会计算为 16px * 1.5 = 24px。

这意味着，rem 更适合用于跨设备、响应式设计中的长度单位，因为它是相对于根元素的大小而计算的，不受父元素的影响。

而 em 更适合于相对大小的定义，例如设置元素的内边距、外边距、边框宽度等，因为它们相对于字体大小来计算。 值得注意的是，对于文本的字体大小，建议使用 rem 单位而不是 em 单位，以避免字体大小层层嵌套导致最终显示效果与预期不符合。

什么叫优雅降级和渐进增强？两者有什么区别？

优雅降级（Graceful Degradation）和渐进增强（Progressive Enhancement）都是 Web 开发中的两种策略，用于处理不同浏览器及网络环境下的网页显示效果问题。

  • 优雅降级：先针对最新的浏览器（如 Chrome、Firefox、Safari）进行开发，而在旧版本浏览器上保证网页基本功能能够正常运行。如果新版本浏览器不支持某些功能，会进行降级处理，以保证网页可用性。即是对不支持现代功能的旧浏览器进行性能上的降级，使网站在旧浏览器上看起来跟新的浏览器看起来差不多。
  • 渐进增强：先针对低版本的浏览器（如 IE8、IE9）开发网页，确保网页能够在低版本浏览器上正常运行。然后再增加一些现代浏览器才支持的功能。即是说，先保证网站在旧浏览器上基本功能能够正常运行，再将一些新特性逐步添加进去，增强网站的功能和表现。

两者的区别在于：

  1. 优雅降级是从先开发现代浏览器，然后逐步向低版本浏览器降级；

  2. 而渐进增强则是从先开发基本功能，逐渐增加现代浏览器支持的高级功能。

无论是优雅降级还是渐进增强，都是为了在不同浏览器及网络环境下，保证网页能够正常运行和良好的用户体验。具体的选择可以根据网站的特点、目标受众、技术难度等因素来进行决策。

什么是外边距重叠？重叠的结果是什么？

外边距重叠指相邻的两个或多个元素之间的 margin 会发生合并（即取最大值或最小值），导致外边距产生重叠的现象。

外边距重叠会造成在某些情况下边距大小不如预期，也可能会对页面布局产生意想不到的影响。 外边距重叠的结果是，如果两个元素的 margin 发生重叠，则 margin 大小将变成两个 margin 中较大的值。如果发生多次重叠，则最终的 margin 大小取最大值。

外边距重叠的情况包括：

  • 相邻兄弟元素的 margin-top 和 margin-bottom 会重叠。
  • 父元素的 margin-top 和第一个子元素的 margin-top 会重叠。同样地，父元素的 margin-bottom 和最后一个子元素的 margin-bottom 也会重叠。
  • 空元素的 margin-top 和 margin-bottom 会重叠。 避免外边距重叠的方法包括：使用 padding 来代替 margin，包裹元素来避免边距重叠，使用 BFC 等方法。

rgba()和 opacity 的透明效果有什么不同？

rgba() 和 opacity 都可以用来设置元素的透明效果，但它们有以下的不同点：

  • rgba()：可以设置元素颜色的 r、g、b、a 四个分量，其中 a 分量表示透明度，取值范围为 0-1。该属性只影响元素本身的透明度，不会影响该元素所包含的内容和子元素的透明度。同时 rgba() 对 background-color、border-color、outline-color 和 text-color 等属性都有效。
  • opacity：表示元素的不透明度，取值范围为 0-1。当设置 opacity 时，其中包含的所有元素和内容都会变得更加透明或者不透明。即 opacity 属性包括了元素本身以及所有子元素和内容的透明度，对整个元素及其内容起作用。同时 opacity 可以用于设置图片、文本等元素的透明度，但对于背景色、边框颜色、outline 颜色等并不起作用。

因此，需要在具体应用时根据实际情况选择使用 rgba() 还是 opacity。如果需要只设置元素本身的透明度，可以使用 rgba()；如果需要同时设置整个元素包括子元素的透明度，可以使用 opacity。同时为了增加浏览器的兼容性，可以同时使用两者来设置透明度。

CSS 中自适应的单位都有哪些？

CSS 中自适应的单位包括：

  • 百分比（%）：百分比相对于父元素的大小来计算，可用于设置宽高、边距等属性。

  • em：相对于当前元素的 font-size 大小来计算，可用于设置字体大小、边距等属性。

  • rem：相对于根元素（即 html 元素）的 font-size 大小来计算，可用于设置字体大小、边距等属性。

  • vw 和 vh：视口宽度和视口高度的比例，分别为 1/100 的视口宽度和高度，可用于设置宽高等属性。

  • vmin 和 vmax：视口宽度和视口高度中的较小者和较大者，可用于设置宽高等属性。

  • ch：相对于当前字体的字符宽度来计算，可用于设置文本框宽度等属性。

以上单位都具有针对不同场景的自适应性，可以根据具体需要选择合适的单位来实现自适应布局效果。其中，rem 是相对于根元素的大小计算的单位，更适合于整个页面的字体、布局等元素的自适应性控制。而 vw、vh 等基于视口大小的单位则可以用来设置响应式布局中的宽高等属性，更加适合于不同设备屏幕上的自适应布局。

display: none 和 visibility: hidden 的区别是什么？

display: none 和 visibility: hidden 都可以用于隐藏元素，但它们的实现和效果却有所不同：

  1. display: none：
  • 设置元素的 display 属性为 none 时，该元素对文档流将完全不可见，不占据任何渲染空间，该元素及其子元素的所有样式都会被禁用。
  • 元素的 display 值从 none 变为其他值时，该元素会重新出现在文档流中，重新占据渲染空间。
  • display: none 的元素不会触发 click、hover、focus 等事件，也不会被屏幕阅读器读取到，所以适用于完全不需要展示的元素。
  1. visibility: hidden：

  • 设置元素的 visibility 属性为 hidden 时，该元素对文档流仍然可见，但是它不会被渲染到屏幕上。元素的占用空间保持不变，但是它不会对其他元素产生影响。

  • 元素的 visibility 值从 hidden 变为其他值时，该元素会重新出现在屏幕上。

  • visibility: hidden 的元素不会触发 click、hover、focus 等事件，但会被屏幕阅读器读取到，因此适用于需要保留该元素占据空间但是不需要展现的元素。

    综上，display: none 是将元素完全从文档流中移除，不占据任何渲染空间；而 visibility: hidden 则是隐藏元素但是仍然保留元素的占用空间。因此，在具体使用时需要根据具体需求来选择使用哪种属性。

:first-child 和 :first-of-type 的区别是什么？

:first-child 和 :first-of-type 都是 CSS 伪类选择器，它们的区别在于：

  1. :first-child 选择的是其父元素的第一个子元素，无论这个子元素是什么类型。它匹配的是某个父元素的第一个子元素，而不管这个元素的类型是什么。
  2. :first-of-type 选择的是其父元素下第一个与指定标签相同的子元素。它匹配属于其父元素的特定类型的第一个子元素，这里的类型不能是 class 或 id 等属性，必须是同种 HTML 标签元素。

因此，:first-child 和 :first-of-type 的主要区别在于它们匹配的元素不同，前者匹配的是第一个子元素，无论类型是什么，而后者匹配的是第一个指定元素类型的子元素。

什么是盒模型？

盒模型是CSS中一种用于描述HTML元素的方式，它把每个元素看作一个矩形盒子，该盒子由四个部分组成：内容区域、内边距区域、边框区域和外边距区域。

这四个部分按顺序从内到外排列，它们的宽度和高度可以通过CSS属性设置。盒模型在网页布局和样式设计中起到了非常重要的作用，理解它可以帮助我们更好地控制元素的尺寸、边距和边框等属性。

Animation 和 transition 的异同是什么？

Animation和transition都是CSS提供的动画效果，但它们的实现方式和应用场景有所不同。

相同点：

  1. 都可以实现元素在某一时间内的过渡效果；
  2. 都有一些控制属性，例如delay、duration、timing-function等。

不同点：

  1. 实现方式不同：transition通过改变元素某个属性的值(从一个状态到另一个状态的变化)来实现过渡效果，而animation则是通过定义一组关键帧(keyframes)来控制元素的具体变化过程；
  2. 应用场景不同：transition通常用于一些简单的过渡动画效果，例如鼠标移入移出时元素的颜色或透明度的改变，而animation则适用于更复杂的动画效果，例如元素呈现很多个不同的状态，或者需要控制多个属性的变化。

总之，transition适用于场景较简单的动画效果，而animation则适用于场景较复杂的动画效果。

传统表格布局与 div ＋ css 布局有哪些区别？

传统表格布局和 div+css 布局在实现网页布局时存在许多差异。以下是它们之间的一些主要区别：

  1. 结构和语义：

传统表格布局是基于 HTML 表格元素（如 <table>、<tr>、<td> 等）构建的。这种布局方式将内容和表现混合在一起，导致 HTML 结构变得复杂且不易理解。而 div+css 布局将结构（HTML）和样式（CSS）分离，使 HTML 结构更简单、清晰，有利于语义化和可维护性。

  1. 可扩展性和灵活性：

div+css 布局可以通过修改 CSS 来灵活地调整布局，而无需更改 HTML 结构。而表格布局在调整布局时，通常需要修改 HTML 结构，增加了维护成本。div+css 布局还可以方便地实现响应式设计，以适应不同设备和屏幕尺寸。

  1. 性能：

表格布局会影响页面的渲染性能。因为浏览器需要等待整个表格的内容加载完成后，才能确定表格的布局和尺寸。而使用 div+css 布局时，浏览器可以逐行渲染页面，无需等待整个页面加载完成。

  1. 可访问性和SEO：

使用 div+css 布局有利于提高网站的可访问性。它能让屏幕阅读器等辅助设备更容易识别页面结构，提高可访问性。而表格布局会增加 HTML 结构的复杂性，影响屏幕阅读器的解析。此外，搜索引擎优化（SEO）也倾向于更加语义化和结构清晰的 HTML，因此 div+css 布局有利于 SEO。

  1. 兼容性：

现代浏览器对 div+css 布局有很好的支持，可以利用更多的 CSS3 功能和布局技术（如 Flexbox、Grid 等）。而表格布局对于一些 CSS3 功能的支持可能较弱，限制了布局的表现能力。

总之，传统表格布局和 div+css 布局之间存在许多区别。如今，div+css 布局已经成为主流的布局方法，因为它具有更好的结构、语义化、灵活性、性能、可访问性和兼容性。当然，在某些特定场景（如展示具有表格结构的数据）中，使用表格布局仍然是合适的。
