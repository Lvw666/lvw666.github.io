JS 函数

什么是函数

思考: 假如我们要分别计算 1 到 10、5 到 12、14 到 35 的整数和

函数就是语句的封装，可以让这些代码方便地被复用

函数具有一次定义，多次调用的优点

使用函数，可以简化问题，让代码更具有可读性

函数的定义和调用

和变量类似，函数必须先定义然后才能使用函数的定义

使用 function 关键字定义函数，function 是功能的意思

函数是功能的封装

函数表达式

函数的调用

执行函数体中的所有语句，就称为调用函数

调用函数非常简单，只需在函数名字后书写圆括号对即可

函数定义之后, 并不会自动执行, 需要调用才可以执行

调用一次, 执行一次, 多次调用就会多次执行

回调函数: 函数能够当作参数进行调用

语句的执行顺序

函数声明提升

和变量声明提升类似，函数声明也可以被提升

函数表达式不能提升

如果函数是用函数表达式的写法定义的，则没有提升特性

变量提升: 只提升定义, 不提升值

函数优先提升(经典面试题)

函数的参数

参数是函数内的一些待定值，在调用函数时，必须传入这些参数的具体值

定义的时候用形参(形式参数, 也叫形参), 调用的时候用实参(实际参数, 也叫实参)

函数的参数, 可有可无, 可多可少，多个参数之间需要用逗号隔开

刚才的累加求和的例子, 就可以使用函数改写

形参和实参个数不同的情况

实参多, 形参少, 多的实参用不上

实参少, 形参多, 没有传值的形参, 结果是 undefined

回顾开头的那个案例

思考: 假如我们要分别计算 1 到 10、5 到 12、14 到 35 的整数和

arguments

函数内 arguments 表示它接收到的实参列表，它是一个类数组对象

类数组对象：所有属性均为从 0 开始的自然数序列，并且有 length 属性

和数组类似可以用方括号书写下标访问对象的某个属性值，但是不能调用数组的方法

函数的所有实参, 都会保存到 arguments 这个类数组中

思考: 计算所有参数的和, 比如fun(1,2,3,4,5), 返回 15, fun(2,3,4), 返回 9

函数的返回值

函数体内可以使用 return 关键字表示“函数的返回值”

return 之后, 代码会返回函数调用处

调用一个有返回值的函数，可以被当做一个普通值，从而可以出现在任何可以书写值的地方

遇见 return 即退出函数

调用函数时，一旦遇见 return 语句则会立即退出函数，将执行权交还给调用者

return 之后的代码不执行

函数如果没有 return, 默认返回 undefined

结合 if 语句的时候，往往不需要写 else 分支了

思考：请编写一个函数，判断一个数字是否是偶数

函数像一个小工厂

题目：喇叭花数是这样的三位数：其每一位数字的阶乘之和恰好等于它本身。

即 abc = a! + b!+ c!，其中 abc 表示一个三位数

试寻找所有喇叭花数。

思路：将计算某个数字的阶乘封装成函数，这样可以让问题简化

JavaScript 内置 sort()方法

数组排序可以使用 sort()方法，这个方法的参数又是一个函数

这个函数中的 a、b 分别表示数组中靠前和靠后的项

如果需要将它们交换位置，则返回任意正数；否则就返回负数

sort 会改变原来的数组, 所以不需要额外声明变量来接收

可以简化代码

递归和循环的关系

能用递归解的都可以用循环来写。

循环比递归的效率高（递归每调用一次都要存一份当前 function 的副本，时间内存都消耗增大）

只有在循环写出来会把人看晕的情况下，优先选递归，递归使得代码《简单易懂》

递归的逻辑

先递进，再回归

编写递归的原则

找临界值, 所谓临界值, 就是无需计算可以知道结果的值

找当前运算和上一次运算的关系

无限递归

当递归没有临界值的时候, 就会变成无限递归, 会导致内存溢出

死循环和无限递归的区别

死循环会导致 CPU 占用率 100%

无限递归会引起内存泄漏, 最终导致内存溢出

内存溢出和内存泄漏

内存泄漏：（Memory Leak）是指程序中己动态分配的内存由于某种原因程序未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃等严重后果

内存溢出：（out of memory）通俗理解就是内存不够，通常在运行大型软件或游戏时，软件或游戏所需要的内存远远超出了你主机内安装的内存所承受大小，就叫内存溢出。

引用类型值的相关知识

JavaScript 中的数据类型有两类：基本类型和引用类型值

深克隆和浅克隆

使用arr1 = arr2的语法不会克隆数组, 因为指向同一个内存空间, 两个变量并没有分开

浅克隆, 只克隆数组的第一层，如果是多维数组，或者数组中的项是其他引用类型值，则不克隆其他层, 藕断丝连

深克隆, 克隆数组的所有层，要使用递归技术

如果遍历到项是基本类型值，则直接推入结果数组；

如果遍历到的项是又是数组，则重复执行浅克隆的操作

变量作用域

JavaScript 是函数级作用域编程语言, 变量只在其定义时所在的 function 内部有意义。

函数里面定义的变量, 我们称之为局部变量

通俗点说: 函数里面的变量, 函数外不能用

全局变量

如果不将变量定义在任何函数的内部，此时这个变量就是全局变量，它在任何函数内都可以被访问和更改

通俗点说: 函数外面的变量, 函数里可以用

思考: 如果函数外定义了一个变量, 函数里定义了一个重名的变量, 会怎样?

如果函数中也定义了和全局同名的变量，则函数内的变量会将全局的变量“遮蔽”

注意考虑变量声明提升的情况

变量声明提升, 会把变量的声明, 提升到当前作用域的最前面

函数嵌套

一个函数内部也可以定义一个函数

和局部变量类似，定义在一个函数内部的函数是局部函数

同理: 函数外的函数(全局), 函数里可以调用, 函数里的函数(局部), 函数外不能调用

作用域链

在函数嵌套中，变量会从内到外逐层寻找它的定义

不加 var 将定义全局变量

在初次给变量赋值时，如果没有加 var，则将定义全局变量

闭包是函数本身和该函数声明时所处的环境状态的组合

函数能够记忆住其定义时所处的环境，即使函数不在其定义的环境中被调用，也能访问定义时所处环境的变量。
