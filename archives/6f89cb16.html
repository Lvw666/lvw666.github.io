什么是语义化？语义化有什么作用？HTML 有哪些语义化标签？

语义化是指：

语义化是指使用有意义的标签和属性来描述页面内容的过程，使得页面的结构、内容和意义明确，便于人类和计算机理解和处理。

语义化的作用包括：

  • 提高页面的可访问性和可用性
  • 提高搜索引擎的优化效果
  • 使代码更易于维护和理解
  • 提高网站的可维护性和可扩展性

HTML 中的一些语义化标签包括：

  • header：页面顶部区域，包含标题、导航等内容；
  • nav：导航链接集合；
  • main：页面主要内容区域；
  • article：独立的文章、帖子等内容；
  • section：页面中的一个区块，可以包含多个 article；
  • aside：与主要内容相关但不是核心的内容，如侧边栏或广告；
  • footer：页面底部区域，包含版权信息、联系方式等内容。

img 标签上的 alt 与 title 有什么区别？

在 HTML 的 img 标签中，alt 和 title 属性都可以用于给图像添加描述，但它们有以下不同之处：

  1. alt 属性用于提供图像的替代文本，即在图像无法显示或者被屏幕阅读器解析时，能够替代图像的文本描述。因此，alt 属性需要提供一个简短、具有代表性且与图像相关的描述。
  2. title 属性用于为图像提供附加信息，鼠标悬浮在图像上时会显示。通常用于向用户提供更多的相关信息，如图片的来源、版权信息等等。
  3. 总之，alt 属性是为了更好的访问性而提供的，title 属性是为了提供一些额外的信息而设计的。其中，alt 属性在 SEO（搜索引擎优化）中也非常重要，因为它可以帮助搜索引擎更好地理解页面内容，提高网站的排名。

strong 和 em 与 b 和 i 有什么区别？

“strong” 和 “em” 是表示文本强调的 HTML 标签，它们的语义略有不同。

  1. “strong” 表示重点内容，”em” 表示强调内容。

  2. 此外，”strong” 的强调程度更强，可以影响文本的整体意义。

  3. “b” 和 “i” 也是表示文本格式的 HTML 标签。它们分别代表加粗和斜体。

  4. 与 “strong” 和 “em” 不同，它们更多地表示视觉效果而非语义含义。

HTML、CSS、JavaScript 三者之间的关系是什么？

HTML、CSS、JavaScript 三者是 Web 前端开发的核心技术，它们的关系是：

  1. HTML（Hypertext Markup Language）是用于构建 Web 页面结构的标记语言，通过使用 HTML 元素来定义 Web 页面的各种内容，如文本、图像、链接、表格等等。
  2. CSS（Cascading Style Sheets）是用于控制 Web 页面样式和布局的样式表语言，通过定义样式规则来描述 HTML 元素的外观和布局，如字体、颜色、位置、大小等等。
  3. JavaScript 是一种脚本语言，可以在 HTML 文档中嵌入脚本代码。通过 JavaScript，可以动态地改变 HTML 和 CSS，实现 Web 页面的交互和动态效果，如表单验证、动态生成内容、事件响应等等。
  4. 因此，HTML、CSS、JavaScript 三者是紧密相关的，通常需要配合使用，从而实现 Web 页面的构建、呈现和交互。HTML 用于定义页面结构和内容，CSS 用于控制页面样式和布局，JavaScript 用于实现页面的交互和动态效果。

HTML5 新增了哪些API？

HTML5 新增了许多有用的 API，以下是一些常用的 HTML5 API：

  1. Canvas API：用于绘制图形和动画。
  2. localStorage 和 sessionStorage API：用于存储和访问本地数据。
  3. Geolocation API：用于获取用户设备的地理位置信息。
  4. Web Workers API：允许在主线程之外创建并运行多个线程，提升页面性能。
  5. Web Storage API：用于在浏览器中存储键值对数据，不同于 Cookie 的特点是可以存储更多数据，并且不会自动发送给服务器。
  6. Web Audio API：提供了创建、控制和处理音频的接口。
  7. WebSockets API：提供了基于 TCP 协议的全双工通信功能，用于实现实时通信、在线游戏和实时交互等功能。

Geolocation API 可以用于获取用户设备的位置信息，从而实现针对不同地理位置的个性化服务。

Web Storage API 可以用于在浏览器本地存储和访问键值对数据，通常用于缓存数据，避免重复查询后端接口等操作，提高页面性能。

HTML5 新增了哪些表单元素？

HTML5 新增了一些在表单中使用的新元素，以下是一些常用的 HTML5 表单元素：

  • <datalist>：定义输入框中可选的选项列表。
  • <keygen>：定义一对密钥，用于客户端和服务器之间的加密通信。
  • <output>：用于在表单中显示计算结果或输出信息。
  • <progress>：用于显示任务的完成进度。
  • <meter>：用于表示已知范围内的标量测量。
  • <color>：用于让用户选择颜色。
  • <date>、<time>、<datetime-local>：分别用于选择日期、时间和日期时间。

HTML5 还引入了一些新的表单属性，如 placeholder 属性、required 属性、autocomplete 属性等，它们进一步增强了表单元素的功能。新增的表单元素和属性使得 Web 表单更加灵活和实用，提高了用户体验和开发效率。

HTML5 为浏览器提供了哪些数据存储方案？

HTML5 为浏览器提供了一些数据存储方案，以下是一些常见的方案：

  1. localStorage 和 sessionStorage：这两种数据存储方式可以在客户端本地存储键值对数据，并且支持非常大的存储容量。localStorage 是在浏览器关闭后也可以保留数据的，而 sessionStorage 只在当前会话（包括页面刷新）中有效。
  2. IndexedDB：IndexedDB 是一种面向对象的数据库，可以在客户端本地存储非结构化数据（像一些 JSON 数据）。相较于 localStorage 和 sessionStorage，IndexedDB 具有更高的存储容量和更好的查询能力，但是学习成本更高，使用也更为复杂。
  3. Web SQL Database：Web SQL Database 基于 SQL 语言，并且兼容常见的 RDBMS 系统，如 SQLite。由于已经停止维护和支持，所以不推荐使用。

这些数据存储方案可以用于在客户端浏览器上存储和访问数据，从而实现某些功能，如本地缓存、离线应用等。同时这些方案具有不同的优势和限制，需要根据不同的需求选择合适的方案。
