router-view

<router-view> 是 Vue Router 的一个组件，它用来渲染当前路由所对应的组件。

可以把 <router-view> 看作是一个占位符，它会根据当前路由自动渲染相应的组件。

在一个 Vue 应用中，应用界面和路由之间的对应关系就是通过 <router-view> 来实现的。

当路由发生改变时，<router-view> 会根据当前的路由，动态地加载并渲染对应的组件。

值得注意的是，<router-view> 也支持嵌套。也就是说，一个被 <router-view> 渲染的组件内部也可以包含自己的 <router-view>，用于渲染子路由组件。这在构建嵌套路由的时候是非常有用的。

router-linker

<router-link> 是 Vue Router 提供的一个组件，用于创建基于应用路由的链接。

可以把 <router-link> 看作是 <a> 标签的增强版。

<router-link> 主要有以下特性：

  1. 默认渲染为 <a> 标签：在默认情况下，<router-link> 会被渲染为 <a> 标签。

  2. 支持命名的路由和参数传递：<router-link> 可以通过 to 属性来指定路由的位置。to 属性的值可以是一个字符串或一个描述地址的对象。

  3. 支持 active 类：当其 to 属性的路由被成功匹配时，<router-link> 会自动添加一个默认的 router-link-active 类。

  4. 支持 exact 属性：如果你只想在当前的路由完全匹配 to 属性时才添加 router-link-active 类，可以使用 exact 属性。

  5. 支持自定义标签：通过 tag 属性，你可以自定义 <router-link> 渲染的标签类型。

同步加载与异步加载路由

在 src\router\index.js 中, 路由有两种定义方式

异步加载路由的含义, 就是只有跳转到指定 url 的时候, 才会加载组件, 不跳转不加载

同步加载路由, 不管是否跳转到指定的 url, 都是加载相关的组件

异步加载路由

优点: 节省网络性能

缺点: 路由切换加载慢

同步加载路由

优点: 路由切换加载快

缺点: 损耗网络性能
