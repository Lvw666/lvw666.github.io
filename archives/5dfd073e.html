不同域和同域

不同域: 协议、域名、端口号，任何一个不一样，就是不同域

tips: 与路径无关，路径是否相同无所谓

https 协议, 默认端口 443

http 协议, 默认端口 80

默认端口可以省略不写

跨域请求

同域: 发送请求的地址, 和响应的地址, 在同一域下

跨域: 发送请求的地址, 和响应的地址, 不在同一域下，就叫跨域

不同域之间的请求, 就是跨域请求

跨域请求会被阻止, 原因如下:

  1. 阻止跨域请求，其实是浏览器本身的一种安全策略 – 同源策略
  2. 当一个 Web 页面发起跨域请求时，浏览器会根据同源策略（同协议、同域名、同端口）禁止该请求，因为这会导致网站的安全问题。
  3. 其他客户端或者服务器都不存在跨域被阻止的问题

跨域解决方案

  1. CORS 跨域资源共享

  2. JSONP

先使用 CORS 跨域资源共享，如果浏览器不支持 CORS 的话，再使用 JSONP

CORS 跨域资源共享

CORS 全称 Cross-Origin Resource Sharing，用于解决浏览器跨域资源共享问题。

CORS 的主要作用是允许一个域上的Web应用可以与另一个域上的资源交互，包括读取、修改和删除等。

需要后端, 在发送响应时, 添加字段

使用 CORS 跨域的过程如下所示:

  1. 浏览器发送请求
  2. 后端在响应头中添加 Access-Control-Allow-Origin 头信息
  3. 浏览器接收到响应
  4. 如果是同域下的请求，浏览器不会额外做什么，这次前后端通信就圆满完成了
  5. 如果是跨域请求，浏览器会从响应头中查找是否允许跨域访问
  6. 如果允许跨域，通信圆满完成
  7. 如果没找到或不包含想要跨域的域名，就丢弃响应结果

CORS 的兼容性: IE10 及以上版本的浏览器可以正常使用 CORS

JSONP 实现跨域

JSONP 全称 JSON with Padding 是一种利用 Script 标签的跨域数据传输方式

Script 标签跨域不会被浏览器阻止

JSONP 跨域解决方案, 只支持 GET 请求

需要服务器端准备好 JSONP 接口, 示例如下:

http://suggest.taobao.com/sug?code=utf-8&q=商品关键字&callback=cb
