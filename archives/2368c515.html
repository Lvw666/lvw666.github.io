什么是事件监听

DOM 允许我们书写 JavaScript 代码以让 HTML 元素对事件作出反应

什么是事件：用户与网页的交互动作

  • 当用户点击元素时
  • 当鼠标移动到元素上时
  • 当文本框的内容被改变时
  • 当键盘在文本框中被按下时
  • 当网页已加载完毕时
  • ……

监听，顾名思义，就是让计算机随时能够发现这个事件发生了，从而执行程序员预先编写的一些程序

设置事件监听的方法主要有 onxxx 和 addEventListener()两种

add 增加

event 事件

listener 监听

常见的鼠标事件监听

mouse 鼠标

click 点击

dblclick 双击

down 按下

up 弹起

enter 进入

leave 离开

move 移动

事件名 事件描述
onclick 在某个对象上, 鼠标单击
ondblclick 在某个对象上, 鼠标双击
onmousedown 在某个对象上, 鼠标按下
onmouseup 在某个对象上, 鼠标抬起
onmousemove 在某个对象上, 鼠标移动
onmouseenter 在某个对象上, 鼠标移入(相似事件, onmouseover)
onmouseleave 在某个对象上, 鼠标移出(相似事件, onmouseout)

onclick

当鼠标单击某个对象

ondblclick

当鼠标双击某个对象

dbl double 双

onmouseup

onmouseup: 当鼠标按键在某个对象上被松开

onmousedown

onmousedown: 当鼠标按键在某个对象上被按下

触发顺序: onmousedown ==> onmouseup ==> onclick

onmousemove

当鼠标按键在某个对象上被移动, 触发频次很高

onmouseenter / onmouseover

onmouseenter/onmouseover: 当鼠标进入某个对象

onmouseleave / onmouseout

onmouseleave/onmouseout: 当鼠标离开某个对象

常见的键盘事件监听

key 按键

press 按下

down 按下

up 弹起

事件名 事件描述
onkeypress 键盘被按下(系统按键无法识别)
onkeydown 键盘被按下(先于 onkeypress 触发, 可以识别系统按键)
onkeyup 键盘被松开

onkeypress

当某个键盘的键被按下（系统按钮如箭头键和功能键无法得到识别）

onkeyup

onkeyup: 当某个键盘的键被松开

onkeydown

onkeydown: 当某个键盘的键被按下(系统按钮可以识别，并且会先于onkeypress发生)

触发顺序: onkeydown => onkeypress => onkeyup

常见的表单事件监听

事件名 时间描述
onchange 改变 域 的内容(input, textarea)
onfocus 元素获得焦点
onblur 元素失去焦点
onsubmit 表单提交的时候
onreset 表单重置的时候

onchange

onchange: 当用户改变域的内容, 离开时触发, 失去焦点的时候触发

oninput

oninput:当用户改变域的内容, 立即触发

onfocus

onfocus: 当某元素获得焦点（比如 tab 键或鼠标点击）

onblur

onblur:当某元素失去焦点

onsubmit

onsubmit: 当表单被提交

onreset

onreset: 当表单被重置

this 的含义

把事件添加到哪个 dom 元素上面, 这个 this 就是哪个 dom 元素
